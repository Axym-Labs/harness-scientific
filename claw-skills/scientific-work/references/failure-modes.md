# Scientific Automation Failure Modes

This checklist distills practical cautions from Yuxi Liu's critique of fully
autonomous research agents and the local reliability requirements.

| Failure mode | Countermeasure |
|--------------|----------------|
| Recombining familiar ideas while presenting them as novel | Search prior work and state the novelty test before claiming contribution. |
| Implementation drift from the intended method | Inspect code/configuration and verify intervention fidelity before interpreting results. |
| Context loss turning uncertainty into a confident story | Preserve hypotheses, decisions, measurements, and caveats in durable notes. |
| Premature success from a convenient metric movement | Require controls, alternatives, and a link from metric to question. |
| Missing domain knowledge or tacit constraints | Surface assumptions and obtain domain review for meaningful claims. |
| Optimizing what is measurable rather than scientifically useful | Evaluate insight, relevance, and information gained, including negative results. |

Source:

- Yuxi Liu, ["Who is Seriously Using Claw Scientist to do Fully Autonomous Research? Please Don't"](https://www.linkedin.com/pulse/who-seriously-using-claw-scientist-do-fully-autonomous-liu-hlmrc/)
